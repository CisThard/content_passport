# Example Asset Patterns

Reference these examples to learn common asset patterns on Sui.

## [Fixed SupplyInitialize a coin with a fixed total supply, then lock the supply to prevent further minting using dynamic fields and freezing.→](</onchain-finance/examples-patterns/fixed-supply>)## [Loyalty TokensUse the Sui Closed-Loop Token standard to create tokens that are only valid within specific workflows and services. One example of Closed-Loop Tokens is a loyalty token.→](</onchain-finance/examples-patterns/loyalty-tokens>)## [In-Game CurrenciesUse the Sui Closed-Loop Token standard to create assets that you can use as currency within a game application.→](</onchain-finance/examples-patterns/in-game-currency>)## [Soulbound NFTsUse Sui Move struct abilities and the Sui framework transfer module to make an NFT soulbound (non-transferable).→](</onchain-finance/examples-patterns/soulbound-tokens>)## [NFT RentalsAn example that enables users to rent NFTs according to the rules of a provided policy instead of owning them outright.→](</onchain-finance/examples-patterns/nft-rental>)## [Kiosk Joining PatternsUse the asset tokenization packages to create fractional NFTs and fungible tokenized assets on Sui, with support for Kiosk-based transfer policies.→](</onchain-finance/examples-patterns/kiosk>)## [WebAssembly and Template PatternThis feature was developed to enable Move bytecode serialization and deserialization on the web. In essence, this feature allows you to edit existing contracts in a web environment.→](</onchain-finance/examples-patterns/wasm-template>)
